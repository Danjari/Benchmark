# Product — SocraticRAG

## What It Is

A benchmark dataset and evaluation framework for measuring whether LLMs can generate pedagogically faithful Socratic responses in a RAG setting. It evaluates three orthogonal dimensions jointly, which no existing benchmark does:

1. **Direct-Answer Leakage** (Metric 1): Does the model give the answer instead of guiding?
2. **Retrieval Faithfulness** (Metric 2): Does the model stay within the retrieved document?
3. **Pedagogical Alignment** (Metric 3): Does the response target the student's specific misconception?

## The Core Artifact

**Edu-QA-Socratic**: a dataset of (C, P, U, R*) tuples where:
- C = a semantically coherent chunk (200-500 tokens) from a university course document, representing a single concept (definition, example, or explanation). C is the model's sole permitted knowledge source.
- P = a student profile encoding one of four cognitive states: accurate, erroneous, comprehension, confusion (from Discerning Minds, Liu et al. 2025)
- U = a student utterance expressing that state
- R* = the Golden Socratic Response: the best Socratic question for that (C, P, U) triple, selected and lightly revised from LLM-generated candidates by expert educators, with supporting sentences in C explicitly highlighted

Target scale: 400-800 gold (C, P, U, R*) items across 50-100 chunks, 5-10 courses, 3-4 subject domains.

## How the Two Source Pipelines Combine

The dataset construction uses two frameworks from different papers, on orthogonal dimensions:

- **Dean-Teacher-Student pipeline** (SocraticLM, Liu et al. 2024): governs the generation process — how candidate student misconceptions and Socratic responses are produced and filtered. The Dean agent rejects pedagogically implausible utterances and utterances that cannot be derived from C.
- **Four-state cognitive taxonomy** (Discerning Minds, Liu et al. 2025): governs the student profile space — what kind of understanding a student expresses. Defines the four states (accurate/erroneous/comprehension/confusion) as two orthogonal axes (answer-correctness vs. metacognitive-expression).

The DTS pipeline generates; the 4-state taxonomy classifies. A student utterance generated by DTS is then categorized by the taxonomy to ensure coverage of all four diagnostic states per chunk.

## The Evaluation Framework

Three automated metrics, each using LLM-as-a-Judge:
- Metric 1: Adapted from EULER's "reveal_answer" criterion (validated at Pearson r = 0.78 against human judgment)
- Metric 2: Novel two-step NLI mechanism: (1) LLM extracts declarative presuppositions from the Socratic question, (2) entailment check against C. Adapted from FActScore (Min et al., 2023) for interrogative outputs.
- Metric 3: GuideEval three-phase rubric (Perception, Orchestration, Elicitation) from Discerning Minds, with Elicitation scored on Bloom's taxonomy E-Levels 1/2/3.

Self-enhancement bias mitigation: cross-judge protocol (Claude judges GPT-4o outputs, GPT-4o judges Claude outputs).

## What Makes It New

1. First benchmark to evaluate retrieval faithfulness + Socratic quality jointly.
2. First faithfulness metric that works on non-declarative (interrogative) outputs.
3. Multi-domain corpus (not math-only like MathTutorBench).
4. Silver-to-gold annotation pipeline validated by professor review.

## Non-Goals

- Not a tutoring system — it is an evaluation tool for tutoring systems.
- Not a dataset for training Socratic models — it is a benchmark for evaluating them.
- Not an end-to-end RAG pipeline — it evaluates the response generation step only.
